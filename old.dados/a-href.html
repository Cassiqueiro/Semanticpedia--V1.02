Expected announcement

  1. "element"
  2. "test label"
  3. "element with title, test description"
  4. "test label"
  5. "test label, test description"

Notes

Latest versions of 3 windows screen readers:

  • JAWS
  • NVDA
  • Window Eyes

Latest versions of 3 browsers:

  • Firefox
  • Chrome
  • Internet Explorer

3 ways to interact with the links were tested:

  • cursor: using the up/down arrow keys
  • nav key: using the screen readers link navigation keys (U/V for JAWS and NVDA, U/L for Window Eyes)
  • tab key: using the tab key

Results Overview

The announcement of label and description for links varies across screen reader, interaction method and browser.

  • All screen readers announce the link label correctly across all browsers independent of interaction method (except NVDA does not announce the label when provided via aria-label (test 2) when Chrome is used).
  • Window Eyes does not announce the description at all, but has the most consistent label announcement across interaction methods and browsers.
  • JAWS only announces the description when a user tabs to a link.
  • NVDA only announces the description when using the link nav keys or tab keys in Firefox and Chrome. It does not announce the description at all with IE.
  • Results in the tables that differ from the expected output are marked
